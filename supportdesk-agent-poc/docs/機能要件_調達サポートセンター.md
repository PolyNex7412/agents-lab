# 調達システム サポートセンター 機能要件

調達システムのサポートセンターで使えるシステムを想定した機能要件です。  
**会話の音声**を起点に、**過去の類似問い合わせの表示** → **1次回答の出力** → **エスカレーション判断**を効率的に実現することを核に、周辺機能を整理しています。

---

## 1. システムの目的・スコープ

| 項目 | 内容 |
|------|------|
| **対象** | 調達システム（購買・発注・承認・検収・ベンダー管理等）の問い合わせ対応 |
| **主な利用者** | サポートセンターのオペレーター（1次対応）、必要に応じて2次対応・管理者 |
| **ゴール** | 音声/テキストから類似事例を素早く提示し、1次回答とエスカレ判断を支援して対応品質・スピードを上げる |

---

## 2. コアフロー（必須要件）

### 2.1 音声を起点とした処理

| 機能ID | 機能概要 | 詳細 |
|--------|----------|------|
| **V1** | **音声入力** | 通話や録音の音声をシステムに入力できる。リアルタイム／録音ファイルのいずれか（または両方）を想定。 |
| **V2** | **音声→テキスト（STT）** | 音声をテキストに変換する。言語は日本語を第一とし、必要に応じて英語・数値・略語の認識を考慮。 |
| **V3** | **発話区間の扱い** | 通話の場合は「発話の区切り」や「誰の発話か」を保持できると、問い合わせ文の切り出しやログ表示に有利。 |

### 2.2 類似問い合わせの検索・表示

| 機能ID | 機能概要 | 詳細 |
|--------|----------|------|
| **R1** | **過去問い合わせとのマッチング** | 入力テキスト（または音声から得たテキスト）と、過去の問い合わせ・FAQ・ナレッジを類似度で検索する。 |
| **R2** | **マッチ順での一覧表示** | 検索結果を**類似度（マッチ順）**でソートして一覧表示する。スコアやランキングを表示できるとよい。 |
| **R3** | **参照元の明示** | 各候補が「過去ログ」「FAQ」「手順書」のどれかを識別できるようにする。 |
| **R4** | **検索対象の範囲** | 調達ドメインに特化したナレッジ（購買申請・承認フロー・PO・検収・ベンダー・見積など）を優先して検索する。 |

### 2.3 1次回答の出力

| 機能ID | 機能概要 | 詳細 |
|--------|----------|------|
| **A1** | **1次回答文の生成** | 類似事例・FAQを根拠に、オペレーターがそのまま／編集して使える**1次回答文**を生成する。 |
| **A2** | **根拠の提示** | どのナレッジ・どの過去事例を参照したかを回答と一緒に表示する（引用・リンク）。 |
| **A3** | **テンプレ・定型文** | よく使う定型文・テンプレートを登録し、意図に応じて挿入・編集できると効率的。 |

### 2.4 エスカレーション判断

| 機能ID | 機能概要 | 詳細 |
|--------|----------|------|
| **E1** | **エスカレ判定の提示** | システムが「要エスカレーション／1次で完結可能」の判断（または確度）を提示する。 |
| **E2** | **判定理由の表示** | 信頼度の低さ・意図不明・回答文に「担当へ」が含まれるなど、判定理由を簡潔に表示する。 |
| **E3** | **エスカレ先の案内** | エスカレ時は、担当部署・窓口・チケット起票先などの案内を表示できるようにする。 |

## 3. 周辺機能要件

### 3.1 問い合わせ・ログ管理

| 機能ID | 機能概要 |
|--------|----------|
| **L1** | 問い合わせの**ログ保存**（日時・内容・意図・1次回答・エスカレ有無・対応者など） |
| **L2** | ログの**検索・フィルタ**（期間・意図・キーワード・エスカレ有無） |
| **L3** | 通話・チャット単位での**セッション管理**（1通話＝1セッションなど） |
| **L4** | 解決済み事例を**ナレッジ化**して検索対象に取り込む運用を想定したデータ構造 |

### 3.2 ナレッジ・FAQ管理

| 機能ID | 機能概要 |
|--------|----------|
| **K1** | **FAQ・手順書**の登録・更新・無効化（調達プロセス・画面別・エラー別など） |
| **K2** | ナレッジの**カテゴリ・タグ**（購買申請・承認・PO・検収・ベンダー等） |
| **K3** | **過去問い合わせの流用**：ログから「よくあるパターン」をFAQ候補として登録する支援 |

### 3.3 ダッシュボード・分析

| 機能ID | 機能概要 |
|--------|----------|
| **D1** | **件数・自動解決率・エスカレ率**などのKPI表示（日・週・月） |
| **D2** | **意図別・カテゴリ別**の件数・傾向 |
| **D3** | **平均対応時間**や**検索ヒット率**などの運用指標（取得できるデータがある場合） |
| **D4** | レポートの**エクスポート**（CSV/Excel） |

### 3.4 オペレーター向けUI

| 機能ID | 機能概要 |
|--------|----------|
| **U1** | **音声入力**（マイク／ファイルアップロード）と**テキスト入力**の両方に対応 |
| **U2** | **類似問い合わせ一覧**をマッチ順で表示し、クリックで詳細・1次回答を表示 |
| **U3** | **1次回答**の表示・コピー・簡易編集 |
| **U4** | **エスカレ判定**と理由の常時表示 |
| **U5** | 通話中は**リアルタイム表示**（音声→テキスト→検索結果の更新）を検討 |

### 3.5 管理者・設定

| 機能ID | 機能概要 |
|--------|----------|
| **M1** | **意図・カテゴリ**のマスタ管理（キーワード・ルールの設定） |
| **M2** | **エスカレ先**・担当部署のマスタ管理 |
| **M3** | **閾値の設定**（信頼度でエスカレとするしきい値、検索件数など） |
| **M4** | **ロール・権限**（オペレーター／管理者／参照のみなど） |

### 3.6 連携・セキュリティ

| 機能ID | 機能概要 |
|--------|----------|
| **S1** | **認証・認可**（SSO／AD連携の検討） |
| **S2** | **監査ログ**（誰が何时に何を参照・操作したか） |
| **S3** | **個人情報・機密**の取り扱い方針に沿ったログ保存・マスキング |
| **S4** | 電話・CTI・チャットとの**連携**（将来：着信とセッションの紐付け） |

## 4. 機能要件一覧（サマリ）

| カテゴリ | 必須（コア） | 推奨（周辺） |
|----------|----------------|----------------|
| **音声** | V1 音声入力、V2 音声→テキスト | V3 発話区間 |
| **類似検索** | R1 マッチング、R2 マッチ順表示、R3 参照元 | R4 調達ドメイン特化 |
| **1次回答** | A1 回答生成、A2 根拠提示 | A3 テンプレ |
| **エスカレ** | E1 判定提示、E2 理由表示 | E3 エスカレ先案内 |
| **ログ** | L1 保存 | L2 検索、L3 セッション、L4 ナレッジ化 |
| **ナレッジ** | （既存FAQ拡張） | K1〜K3 |
| **ダッシュボード** | D1 KPI | D2〜D4 |
| **UI** | U1〜U4 | U5 リアルタイム |
| **管理** | — | M1〜M4 |
| **連携・セキュリティ** | — | S1〜S4 |

## 5. 現行PoCからの拡張イメージ

| 現状 | 拡張後のイメージ |
|------|------------------|
| テキスト入力のみ | **音声入力**（STT）→ 同じフローへ |
| 問い合わせ→回答の1本 | **類似過去問い合わせ一覧**をマッチ順で先に表示 → 選択して1次回答 |
| FAQ検索と問い合わせが別タブ | **1画面で**「音声/テキスト → 類似一覧 → 1次回答 → エスカレ判定」を一気に表示 |
| ログは一覧のみ | **ログを検索対象に含め**「過去の類似問い合わせ」としてマッチ順表示 |
| 意図はキーワードベース | 意図マスタ・閾値の設定画面、エスカレ先マスタ |

---

## 6. 非機能・今後の検討

- **パフォーマンス**: 音声→テキスト→検索→回答まで数秒〜十数秒以内を目安に設計。
- **可用性**: サポートセンター稼働時間に合わせたSLA・障害時の代替手段。
- **音声品質**: 雑音・複数話者・方言への対応（STTの選定・チューニング）。
- **多言語**: 海外拠点がある場合は英語等の検索・回答の検討。
